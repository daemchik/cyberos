# CyberOS — Отчёт о состоянии проекта

## Стек

- **Фронт:** React 19, TypeScript, Vite, Tailwind (`cyberos/`)
- **Бэк:** Django 5, MySQL (`backend/`), API на порту **8000** (прокси Vite → `/api`)
- **Демо-данные:** `python manage.py seed_demo` (из каталога `backend/`, после применения `schema.sql`)

## Общий статус: ~90% готовности

---

## Полностью работает (подключено к API)

### Dashboard
- Выручка, ПК, сессии, продажи, задачи — из БД
- Тренд выручки и сессий — сравнение с вчера (`revenueTrendPct`, `sessionsTrendPct`)

### База клиентов, тарифы, промокоды, задачи, маркет, смены, настройки, карта зала
- Как ранее, через REST API Django

### Аналитика и отчёты
- Выручка, продажи, сессии, новые клиенты, топ клиенты, график по дням — из БД
- Пиковые часы — из `sessions` (hourlyLoad)
- Структура по зонам — из `sessions` + `zones` (zoneRevenue)
- Детализация по дням — `dailyDetail` (выручка продаж, сессии, новые клиенты, относительная «загрузка»)
- Тренды на KPI-карточках — по последним двум дням в `dailyRevenue` / `dailyDetail` (продажи: «—», пока нет помесячных чеков)

### Маркет — вкладка «Заказы»
- Заказы игроков — из БД (`player_orders`)

### Карта зала
- Прервать сеанс — `endSession` + освобождение ПК
- Пауза / продолжить — `pauseSession` / `resumeSession` (при наличии строки сессии в БД)
- Добавить время — `extendSession` (минуты через prompt)
- Переименование ПК — сохранение через API, один toast об успехе

### Player (`#/player?ws=<id>`)
- `ws` — `id` рабочей станции в таблице `workstations` (по умолчанию 1)
- Вход / регистрация, баланс и `total_sessions` из БД
- Старт сеанса — `createSession`, списание с баланса на сервере, таймер по секундам
- Пауза / завершение — API сессий
- Пополнение — `updateClient`
- Маркет — `createOrder` со списанием баланса
- «+1 час» — списание 100 ₽ + `extendSession` на 60 мин
- Промокоды — начисление bonus/fixed на баланс в БД

### Ядро системы
- Тумблеры безопасности и уведомления сохраняются в `settings` (ключи `sec_*`, `notifications`)
- Сброс возвращает и значения безопасности по умолчанию

## Частично / заглушки

- **PDF / Excel** в отчётах — по-прежнему toast без файла
- **Player:** «Рабочий стол», «Вызвать администратора» — toast (нет агента на ПК)
- **Карта зала:** WoL, скриншот, удалённый доступ, «перезагрузка» — имитация; таймер «осталось» в панели ПК без синхронизации с `sessions`
- **Светлая тема** — флаг в настройках, без полной смены темы в UI
- **Хранилище / сеть** в ядре — статичные подсказки
